# Llama 3.1 RoPE

该目录实现与 TileOps `RopeLlama31Op` 对齐的 Ascend TileLang 算子。公开接口
只接收 `x`，Llama 3.1 cos/sin 由算子内部生成并按 NPU device 缓存。

```python
import torch

from examples.posembedding.rope_llama31 import RopeLlama31Op

op = RopeLlama31Op(
    seq_len=8192,
    head_dim=128,
    dtype=torch.float16,
    layout="2d",
    batch=1,
    num_heads=32,
)
out = op(x)
```

支持范围：

- `layout="1d"`：`x.shape == (S, D)`；
- `layout="2d"`：`x.shape == (B, S, N, D)`，即 BSND；
- dtype：FP16、BF16、FP32；
- D 为正偶数，旋转覆盖完整 D；
- 非连续 x 会先转为连续 Tensor；
- shape、dtype、layout 和频率参数在构造时静态绑定。

旋转固定为 NeoX split-half：

```text
H = D / 2
y[..., :H] = x[..., :H] * cos - x[..., H:] * sin
y[..., H:] = x[..., H:] * cos + x[..., :H] * sin
```

cos/sin 的物理形状为 `(S,H)`，不会在 GM 中展开到 batch、head 或完整 D。
默认频率参数为 `base=10000`、`scale_factor=8`、`low_freq_factor=1`、
`high_freq_factor=4`、`original_max_position=8192`。

不支持 interleaved、TND/packed、BNSD、外部 cos/sin、position IDs、position
offset、部分 rotary dimension 或动态 shape。

FP16/BF16 共用显式 FP32 中间计算路径，FP32 直接计算。两类 dtype
都根据输入选择 blocked 或 table-resident 调度，统一处理 1D/BSND 和
aligned/padded 输入。

## 运行示例

在当前目录直接运行默认 FP16 BSND 示例：

```bash
python rope_llama31.py
```

也可以指定布局、形状和数据类型：

```bash
python rope_llama31.py \
  --layout 1d --seq-len 8192 --head-dim 128 --dtype bfloat16
```

示例会生成 CPU FP32 参考结果，校验 NPU 输出，并打印 selector 选择的 route、
token/head 分块和 BN group。运行需要可用的 TileLang Ascend 环境和 NPU。
